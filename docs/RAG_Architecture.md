# Agnes RAG Architecture

## Overview

Agnes 2.0 adds a Retrieval-Augmented Generation (RAG) layer that grounds every Gemini compliance decision in real, scraped regulatory knowledge. Instead of relying solely on the LLM's training data, each substitutability evaluation now retrieves the most relevant regulatory documents from a local knowledge base, injects them as context into the prompt, and requires the LLM to cite its sources in the evidence trail.

This directly addresses the top hackathon judging criteria:
- **External data sourcing** — 20 real regulatory documents scraped from FDA, USP, NSF, IFANCA, Non-GMO Project
- **Evidence trails** — each fact in the evidence trail cites `[Source]` brackets
- **Hallucination control** — LLM is grounded in retrieved context, not just training data
- **Trustworthiness** — every retrieval is logged; RAGAS-lite scores measure quality
- **Scalability** — `rag_engine.py` is a clean module with a documented production upgrade path

## Index Architecture

### FAISS HNSW Vector Index
- **Embedding model**: `sentence-transformers/all-MiniLM-L6-v2` (384-dim)
- **Index type**: `faiss.IndexHNSWFlat` with `M=32`
- **efSearch**: 64 (higher recall at modest latency cost)
- **Normalization**: L2-normalized embeddings (inner product = cosine similarity)
- **Corpus**: title + source + content concatenated per document

### BM25 Keyword Index
- **Library**: `rank_bm25.BM25Okapi`
- **Tokenizer**: lowercase regex `[a-z0-9]+` (matches regulatory term patterns)
- **Use case**: exact certification names (`USP`, `GMP`, `Halal`), regulatory codes (`21 CFR 111`, `DSHEA`)

### Hybrid Search Formula
```
combined_score(doc) = α × vector_score(doc) + (1−α) × bm25_score(doc)
```
- `α = 0.65` (vector-dominant — better for semantic paraphrases)
- Vector score normalized to `[0, 1]` before fusion
- BM25 score normalized by max BM25 score in corpus
- Top-5 candidates returned from combined ranking

## Retrieval Pipeline (Cell 5 — per evaluation)

```
Query construction
    ↓
    "vitamin-d3 vitamin-d3-cholecalciferol pharmaceutical
     USP GMP Halal Kosher compliance dietary supplement"
    ↓
hybrid_search(index, query, top_k=5)
    ↓
    [vector search FAISS HNSW]  +  [keyword search BM25]
    → top-5 candidate documents with rag_score
    ↓
rerank(query, candidates, top_n=3)
    ↓
    [cross-encoder: cross-encoder/ms-marco-MiniLM-L-6-v2]
    → top-3 documents with rerank_score
    ↓
format_context_block(docs)
    ↓
    "=== REGULATORY CONTEXT ==="
    [1] USP Pharmaceutical Grade vs Food Grade | Source: USP ...
    [2] USP Cholecalciferol Monograph           | Source: USP ...
    [3] FDA 21 CFR Part 111                     | Source: FDA ...
    ↓
Gemini prompt injection
    ↓
    system_instruction = context_block + AGNES_SYSTEM_PROMPT + grounding_rules
    ↓
Gemini response (JSON)
    ↓
    evidence_trail now includes [USP], [FDA], [NSF] citations
    sources_cited[] added to result
    ↓
store_decision(KB/decisions.json)
    ↓
    RAG quality evaluation: faithfulness, answer_relevance, context_recall
```

## Grounding Rules Injected into System Prompt

```
GROUNDING RULES (RAG context provided above):
- The REGULATORY CONTEXT section contains real documents retrieved from the Agnes knowledge base.
- PREFER grounding your evidence_trail items in these documents.
- Cite sources in square brackets, e.g.: "[USP Verification Program] USP-verified products must contain..."
- If a retrieved document is not relevant to a specific point, you may use your own knowledge
  but do NOT fabricate citations.
- The PRECEDENT CASES section (if present) shows similar past evaluations.
  Use them as reference but evaluate the current case independently.
```

## Historical Decision Memory

Every verdict from Cell 5 is persisted to `KB/decisions.json`:

```python
store_decision({
    "ingredient_a":     "vitamin-d3",
    "ingredient_b":     "vitamin-d3-cholecalciferol",
    "supplier_a":       "Prinova USA",
    "supplier_b":       "Prinova USA",
    "grade_a":          "food",
    "grade_b":          "pharmaceutical",
    "certifications_a": ["GMP", "Kosher"],
    "certifications_b": ["USP", "GMP", "Halal", "Kosher"],
    "verdict":          "APPROVE",
    "confidence":       0.98,
    "reasoning":        "Pharma-grade upgrade from food-grade...",
    "evidence_trail":   ["[USP] Pharmaceutical grade meets higher purity spec."],
    "sources_cited":    [{"source": "U.S. Pharmacopeia (USP)", ...}],
})
```

On subsequent evaluations, `retrieve_similar_cases()` uses Jaccard token similarity to find the top-2 most similar past decisions. These are injected as a "PRECEDENT CASES" block, giving the LLM cross-evaluation consistency.

## RAGAS-lite Quality Metrics (Cell 12-RAG)

Three metrics measured per evaluation — no external RAGAS library required:

| Metric | Measurement Method | Formula |
|--------|--------------------|---------|
| **Faithfulness** | Evidence trail items grounded in retrieved docs | `items with ≥20% token overlap / total items` |
| **Answer Relevance** | Verdict aligned with approve/reject signals in docs | `signal keyword count / total signal count` |
| **Context Recall** | Retrieved docs relevant to the query | `docs with ≥10% query token overlap / total retrieved` |
| **Overall** | Mean of three metrics | `(F + AR + CR) / 3` |

### Sample Output
```
Evaluation                    Faithfulness  Answer Relevance  Context Recall  Overall
Eval 1: Supplier Consolidation    100.0%           55.0%          33.3%        62.8%
Eval 2: Cross-Cluster Grade       100.0%           50.0%          33.3%        61.1%
```

### Production Upgrade Path for RAGAS Metrics
- **Faithfulness** → NLI model (`cross-encoder/nli-MiniLM2-L6-H768`)
- **Answer Relevance** → LLM judge (ask Gemini to rate 1–5)
- **Context Recall** → Human-annotated gold-standard relevance dataset
- **Framework** → `pip install ragas` for full RAGAS pipeline

## `rag_engine.py` Public API

```python
from rag_engine import (
    build_index,               # Load KB → build FAISS HNSW + BM25 index
    hybrid_search,             # α·vector + (1-α)·BM25 retrieval
    rerank,                    # Cross-encoder re-ranking
    store_decision,            # Append verdict to KB/decisions.json
    retrieve_similar_cases,    # Jaccard BM25 over past decisions
    format_context_block,      # Format docs as LLM prompt context
    format_precedent_block,    # Format past decisions as LLM precedent block
    evaluate_rag_quality,      # RAGAS-lite scoring
)
```

## Production Upgrade Path

| Hackathon (current) | Production (web app) |
|---------------------|----------------------|
| `faiss-cpu` local index | Qdrant Cloud / Weaviate |
| `all-MiniLM-L6-v2` (384-dim) | `text-embedding-3-small` (OpenAI) |

| `KB/decisions.json` flat file | PostgreSQL `pgvector` table |
| `cross-encoder/ms-marco-MiniLM-L-6-v2` | Cohere Rerank API |
| `scrape_kb.py` on-demand | Scheduled nightly scraper (Playwright + LLM PDF extraction) |
| 20 regulatory documents | 500+ documents (full CFR sections, USP monographs, supplier CoAs) |
| Heuristic RAGAS-lite | Full RAGAS framework with LLM judge |

The key architectural property: **`rag_engine.py` contains zero notebook-specific code**. Swapping the production backends requires only changing the internals of `build_index()` and `hybrid_search()` — the public API stays identical.

## Model Deployment: Cloud + Local Hybrid

Agnes uses a hybrid architecture to balance reasoning quality with resource efficiency:

### Cloud Models (Google GenAI API)

| Model | Usage | Location |
|-------|-------|----------|
| **Gemini Flash** (`gemini-flash-latest`) | Compliance evaluation, multimodal extraction (CoA images, audio, video, PDFs), health assessment | Cloud via `genai.Client(api_key=...)` |

Requires `GEMINI_API_KEY` from `.env` and internet connection.

### Local Models (Cached Offline)

| Model | Usage | Location | RAM Usage |
|-------|-------|----------|-----------|
| **all-MiniLM-L6-v2** (384-dim) | Sentence embeddings for RAG vector search | `models/all-MiniLM-L6-v2/` (~90 MB) | ~400-600 MB |
| **ms-marco-MiniLM-L-6-v2** | Cross-encoder reranking of retrieved docs | `models/cross-encoder-reranker/` (~90 MB) | ~300-500 MB |
| **FAISS HNSW** | Vector similarity search | In-memory + disk index | Minimal |
| **BM25 Okapi** | Keyword retrieval | In-memory | Minimal |

**Total local RAM usage**: ~700 MB - 1.1 GB (both models loaded)

### Caching Local Models

Run once after cloning or when models/ is missing:

```bash
python download_models.py
```

This downloads ~175 MB total into `models/` so Agnes runs fully offline with no HuggingFace network calls.

### Architecture Flow

```
User Input (text/image/audio/video/PDF)
         ↓
┌─────────────────┐     ┌──────────────────┐
│  Gemini Vision  │────→│ Compliance data  │  ←── Cloud
│  (Cloud API)    │     │ extraction       │
└─────────────────┘     └──────────────────┘
         ↓
┌─────────────────────────────────────────┐
│  RAG Retrieval (Local)                  │
│  • FAISS + BM25 hybrid search           │
│  • Cross-encoder rerank                 │
│  • Precedent case retrieval             │
└─────────────────────────────────────────┘
         ↓
┌─────────────────┐     ┌──────────────────┐
│  Gemini Flash   │←────│ Final evaluation  │  ←── Cloud
│  (Cloud API)    │     │ with RAG context  │
└─────────────────┘     └──────────────────┘
```

**Design rationale**: Heavy reasoning (Gemini) stays in the cloud for quality; fast retrieval (embeddings + search) runs locally for speed and offline resilience.
